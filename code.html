Code Walkthrough

This section provides a structured explanation of the simulation implementation. The program is organized into logically distinct stages that directly correspond to the mathematical formulation of the model: parameter definition, system initialization, force evaluation, time evolution, and visual rendering.

1. Parameter Definition

All simulation parameters are collected within a configuration object. These parameters define both the physical behaviour of the particle system and its visual representation.

const settings = { numColors: 4, atomsPerColor: 250, interactionRadius: 80, viscosity: 0.6, timeScale: 1.0, gravity: 0.0, maxSpeed: 4, glow: true, trails: true, showEnergy: true };

These quantities correspond to the mathematical parameters:

  • Interaction radius → \( R_s \)
  • Viscosity (damping factor) → \( \eta \)
  • Time scale → \( \Delta t \)
  • Gravity → external acceleration term

2. System Initialisation

During initialisation, two components are generated: the species interaction matrix and the initial particle states. The interaction matrix assigns a coefficient in the interval [-1, 1] to each ordered pair of species.

function initRules() { for (let i = 0; i < settings.numColors; i++) { rules[i] = []; for (let j = 0; j < settings.numColors; j++) { rules[i][j] = Math.random() * 2 - 1; } } }

Particles are then placed at uniformly random positions within the domain, with their initial velocities set to zero.

3. Force Evaluation

For each unordered pair of particles, the Euclidean separation distance is computed. If this distance is less than the interaction radius, a distance-dependent force is applied.

const dx = b.x - a.x; const dy = b.y - a.y; const dist2 = dx * dx + dy * dy; if (dist2 < r * r) { const dist = Math.sqrt(dist2); const strength = rules[a.color][b.color] * (1 - dist / r); }

This implements the force relationship

\[ F \propto A_{ij} \left( 1 - \frac{d}{R} \right) \]

where the interaction strength decays linearly with distance and vanishes at the interaction boundary. Newton’s third law symmetry is enforced by applying equal and opposite forces to both particles.

4. Velocity Update

After computing the net force on each particle, velocities are updated according to the damped integration rule.

a.vx *= (1 - settings.viscosity); a.vy *= (1 - settings.viscosity); a.vy += settings.gravity;

The viscosity term introduces controlled energy dissipation, ensuring numerical stability and preventing unbounded acceleration. An optional gravitational term may also be included.

5. Position Update

Positions are updated using explicit time stepping:

a.x += a.vx * settings.timeScale; a.y += a.vy * settings.timeScale;

Reflective boundary conditions are applied to ensure particles remain within the simulation domain.

6. Energy Measurement

The total kinetic energy of the system is monitored as:

\[ E = \frac{1}{2} \sum_{i=1}^{N} \left( v_{x,i}^2 + v_{y,i}^2 \right) \]

This provides a global indicator of the dynamical state of the system.

7. Rendering

The particle states are visualized using the HTML5 Canvas API. Rendering is entirely separate from the physical simulation and serves only to display the system evolution in real time.

ctx.beginPath(); ctx.arc(a.x, a.y, 2, 0, Math.PI * 2); ctx.fill();

Computational Considerations

The force evaluation stage scales as \( O(N^2) \), since all particle pairs are considered. For large systems, spatial partitioning techniques such as grid-based neighbor search could significantly reduce computational cost.
